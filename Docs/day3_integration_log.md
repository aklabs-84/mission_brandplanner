## 3. 이중 독립 모달 시스템 정합성

* **담당자**: 서기 (Scribe) 및 해결사 (Solver)
* **목적**: 기존 Stage 3(Few-shot 워딩 튜닝)이 단순 객관식 교정 문제에 머물렀던 한계를 극복하고, 퓨샷(Few-shot)의 실무적 정의인 **"복수 개의 Bad/Good 예시 구조를 통해 원하는 출력 패턴을 유도하고 튜닝하는 기법"**에 맞게 게임의 내용과 UI 구성을 수정하여 실무 학습 가치를 제고함.

### 5.1. 수정 및 구현 내용 (Modifications)
- **퓨샷 학습용 예시 컨테이너 추가 (HTML)**:
  - Stage 3 화면에 `#stage3-shots-container`와 `#stage3-shots-list` 마크업 영역을 신설하여, 문제마다 2개의 Bad ➡️ Good 예시(Few-Shots)가 시각적으로 표시되도록 레이아웃 구축.
- **STAGE3_DEFS 데이터셋 개편 (JS)**:
  - 각 문제마다 학습용 예시 데이터 `shots` 배열을 보강하여 퓨샷 튜닝을 유기적으로 인지하도록 변경.
  - 퀴즈 내용(제품 정의 퓨샷 튜닝, 타겟 고객 퓨샷 튜닝, 예상 성과 퓨샷 튜닝)을 각각 퓨샷 튜닝의 실제 패턴(정량 지표 수치화, 타겟 뾰족화, 성과 구체화)에 어긋나지 않는 보기를 갖도록 수정.
- **자바스크립트 질문 노출 함수 보완**:
  - `showStage3Question()` 함수 내에서 `shots` 배열 데이터를 순회하며 예시 컨테이너에 동적으로 예시 카드들을 렌더링하고, 정답 여부와 보충 코칭 피드백도 퓨샷 원칙에 맞추어 보강 적용.
